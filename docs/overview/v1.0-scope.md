# v1.0 Scope

This document is the canonical reference for planned `v1.0.0` scope and non-goals.

`v0.5.0` does not ship the full `v1.0` scope. It stays focused on scope lock, contract freeze, compatibility policy, and observability hardening so `v1.0.0` can land on a stable base.

## Competitive context

KDCO has set current expectations for third-party OpenCode multi-agent orchestration, especially around delegation and packaging. `v1.0.0` must be competitive there without abandoning OpenCode Council's governance-first differentiation.

## Current reality

- Deliberative orchestration is already shipped and differentiated.
- One-command install is already shipped via `opencode-council`.
- Supervisor feasibility is confirmed by existing OpenCode session primitives.
- The remaining supervisor risk is implementation quality, not platform capability.

## In scope for `v1.0.0`

- Deliberative orchestration model with heartbeat phases, mention-driven routing, and MCP gating.
- One-command install with a fast zero-to-working setup path.
- Usable supervisor mode built on the existing foundation and child-session orchestration.
- Async delegation primitive with governance integration for background research and exploration.
- Governance stack: policy profiles, budget escalation, approval gates, reason codes, audit trail, and evidence packets.
- Contract stability across the public API surface.
- Docs, install guides, and competitive positioning that clearly explain the offer.

## Explicit non-goals for `v1.0.0`

- Building an OCX package manager.
- Replicating terminal auto-spawning or terminal fleet UX.
- Building a visual configurator or web UI.
- Bundling MCP servers.
- Shipping write-capable async background agents.
- Pulling supervisor implementation into `v0.5.0` now that feasibility is confirmed.
